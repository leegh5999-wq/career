# 경력 관리 웹앱 — 기획서

## 한 줄 요약
전기·통신 설계 엔지니어를 위한 **프로젝트 중심 성과·경력 로그**. 평소 가볍게 기록하고, AI가 경력기술서·이력서·성과평가 자료로 정리해 준다.

## 해결하는 문제
평소엔 바빠서 성과를 안 남기다가, 이직·성과평가·협회 경력신고 시점에 "내가 뭘 했더라" 하고 기억을 쥐어짜는 문제. **입력은 가볍게, 출력은 필요할 때** 꺼내 쓰는 구조로 이 공백을 없앤다.

## 타겟
1차 사용자는 본인(전기·통신 설계 엔지니어). 개인용으로 시작하되, 나중에 서비스화할 수 있는 구조로 설계한다.

## 경력 용도 (확정)
- 이직·연봉협상 — 이력서·경력기술서 출력
- 사내 성과평가 — 기간별 성과 근거 자료
- 협회 경력신고·공식 증빙 — 발주처·사업규모·참여기간·역할 등 규정 지향 필드, 기술등급·경력기간 관리

## 핵심 개념
기록 단위는 **프로젝트(현장)**. 설계 엔지니어의 경력은 "어떤 현장에서, 어떤 설비를, 어떤 역할로 설계했나"로 쌓이기 때문이다. 프로젝트 아래에 성과 로그를 붙이고, AI가 이를 정량화된 문장과 경력기술서 초안으로 변환한다.

## 데이터 모델 (초안)
- **Project** — 프로젝트명, 발주처, 용도(공동주택·오피스·플랜트 등), 사업규모(연면적·수전용량·세대수), 역할, 소속사, 설계기간, 상태
  - designScopes(담당 설계범위, 다중) — 수변전 / 간선·동력 / 조명 / 접지·피뢰 / 예비전원 / 소방전기 / 구내통신 / 방송공동수신 / 정보통신설비 / CCTV·방범
  - deliverables — 도면 수량, 계산서 종류(부하·전압강하·조도), 사용 툴(AutoCAD·Revit)
- **Achievement** (성과 로그) — projectId, 원문 메모, AI 정리 문장, 정량 수치·단위, 분류(비용절감·공기단축·품질·안전·기술개선), 날짜
- **Certification** — 자격·교육명, 구분, 발급기관, 취득일, 만료일
- **CareerSummary** — 프로젝트 참여기간 합산·기술등급 산정 보조 (협회 신고용, 대체로 파생값)

## 기능 범위 (Phase)
- **Phase 0** — 기반 셋업 (프로젝트 생성, DB, 데이터 모델, 레이아웃)
- **Phase 1** — 프로젝트 경력 관리 (등록·목록·상세, 담당 설계범위) ← 핵심
- **Phase 2** — 성과 로그 (프로젝트별 한 줄 기록, 분류·정량 수치)
- **Phase 3** — AI 정리 (메모 → 성과 문장, 프로젝트 요약, 경력기술서 초안)
- **Phase 4** — 출력물 (경력기술서·이력서·성과평가 자료 내보내기)
- **Phase 5** — 자격·기술등급 (자격/교육 이력, 경력기간 합산)

MVP = Phase 0~2(구조화된 기록) + Phase 4(비-AI 출력). Phase 3(AI 정리)는 API 키가 준비되면 착수.

## 기술 스택 (확정)
- Next.js(App Router) + TypeScript + Tailwind CSS — 프론트·백엔드 단일 프레임워크
- DB: PostgreSQL(Supabase) + Prisma — 클라우드 저장으로 여러 기기·자동 백업, 계정 확장 용이
- AI: Claude API(claude-sonnet-5) — 서버 라우트에서 호출. **API 키 미준비로 Phase 3은 후순위**, 기록·출력을 먼저 만든다.
